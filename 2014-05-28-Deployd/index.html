Rapid prototyping with Architect and Deployd

A quick way to get applications into a useable state with a rich Node.js backend API, automated documentation, and Sencha Architect.
By: Josh Souza

The goal

  • Architect is useful for rapid prototyping
  • Building a real backend for Architect to talk to can be slow and delays prototyping
  • Setting 'data' in stores leads to a lot of rework, and doesn't allow for 'live' prototypes
  • When you are done with a prototype, it would be helpful to have clearly defined API endpoint needs and datastructures
  • Presenting a prototype should have that 'wow' factor

Technology building blocks

  • Vagrant - Local environment provisioning
  • Sencha ExtJS (Though Touch would be just fine)
  • Sencha Architect - Visual prototyping
  • Deployd - Visual data modelling and API creation
  • Swagger - Visual API documentation

How it works

  • Preconfigured Vagrant app: https://github.com/ShadowZero3000/deployd-vagrant-template
  • Simple initialization of application once set up
  • Creation of Architect project
  • Creation of Deployd data structures when needed
